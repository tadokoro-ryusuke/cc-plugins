# 対話型作業指示作成

## 概要

$ARGUMENTS に基づいて、対話的に作業指示を作成します。

私は以下の項目について順番に質問していきます：

### 1. 基本情報

- **タスクタイトル**: 簡潔で明確なタイトル
- **概要**: 何を実現したいか（1-2 文）
- **背景**: なぜこの機能が必要か

### 2. 機能要件

- **ユーザーストーリー**: 「〜として、〜したい、なぜなら〜」形式
- **受け入れ条件**: 機能が完成したと判断できる条件

### 3. シナリオ（BDD 形式）

以下の形式でシナリオを作成します：

```gherkin
Scenario: [シナリオ名]
  Given [前提条件]
  When [アクション]
  Then [期待結果]
```

### 4. 技術要件

- **影響範囲**: どのモジュール/機能に影響するか
- **API 設計**: 必要なエンドポイントやインターフェース
- **データモデル**: 新規/変更が必要なエンティティ

### 5. 制約事項

- **パフォーマンス要件**: レスポンスタイム、同時接続数など
- **セキュリティ要件**: 認証・認可、データ保護
- **その他の制約**: ブラウザ対応、外部統合など

### 6. 優先度とスケジュール

- **優先度**: High/Medium/Low

## 出力形式

最終的に以下の形式でドキュメントを生成します：

```markdown
# [タスクタイトル]

## 概要

[概要を記載]

## 背景

[背景を記載]

## ユーザーストーリー

[ユーザーストーリーを記載]

## 受け入れ条件

- [ ] 条件 1
- [ ] 条件 2
- [ ] 条件 3

## シナリオ

[BDD 形式のシナリオを記載]

## 技術要件

### 影響範囲

[影響範囲を記載]

### API 設計

[API 設計を記載]

### データモデル

[データモデルを記載]

## 制約事項

### パフォーマンス要件

[パフォーマンス要件を記載]

### セキュリティ要件

[セキュリティ要件を記載]

## 優先度とスケジュール

- 優先度: [High/Medium/Low]
```
