# セッション学習

セッション中の作業パターンを分析し、再利用可能な知識として保存します。

## 実行フロー

### 1. セッション分析

現在のセッションで行った作業を分析：

- 使用したコマンド
- 作成/変更したファイル
- 解決した問題
- 適用したパターン

### 2. パターン抽出

繰り返し使用されたパターンを特定：

- コード構造
- 解決アプローチ
- ベストプラクティス

### 3. スキル保存

抽出したパターンをスキルとして保存：

```
.claude/learnings/
├── 2024-01-20-auth-pattern.md
└── 2024-01-21-api-error-handling.md
```

## 出力形式

```
【セッション学習】

📊 セッション分析:
   実行コマンド: 15
   変更ファイル: 12
   コミット: 5

🔍 抽出されたパターン:

1. 【認証フロー実装パターン】
   - SSO 統合の実装手順
   - トークン管理のベストプラクティス
   - エラーハンドリング戦略

2. 【API エラーハンドリング】
   - 統一エラーレスポンス形式
   - リトライロジック
   - ログ記録パターン

3. 【テスト構造化】
   - AAA パターン（Arrange-Act-Assert）
   - モック戦略
   - カバレッジ最適化

💾 保存先:
   .claude/learnings/2024-01-20-session.md

【推奨アクション】
- 抽出したパターンをチームで共有
- プロジェクト固有のスキルとして登録
```

- プロジェクト: Next.js + Auth.js

## パターン

### 1. トークン管理

```typescript
// トークン更新のベストプラクティス
async function refreshToken() {
  // 実装...
}
```

### 2. エラーハンドリング

...

## 適用条件

- SSO 統合が必要な場合
- Auth.js を使用している場合
```

## 使用例

```bash
# パターン抽出
/dev-core:learn --extract

# 抽出して保存
/dev-core:learn --save

# 過去の学習を表示
/dev-core:learn --list
```
