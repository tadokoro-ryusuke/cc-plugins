# Continuous Learning

## セッション学習の概念

セッション中に行った作業から再利用可能なパターンを抽出し、将来の開発に活用します。

## 学習プロセス

### 1. セッション分析

現在のセッションで行った作業を分析：

- 使用したコマンド
- 作成/変更したファイル
- 解決した問題
- 適用したパターン

### 2. パターン識別

繰り返し使用されたパターンを特定：

- コード構造パターン
- 問題解決アプローチ
- エラー対処方法
- ベストプラクティス

### 3. 知識抽出

抽出した知識を構造化：

```markdown
# パターン名

## コンテキスト

- いつ使用するか
- 前提条件

## パターン

- 具体的な実装
- コード例

## 適用例

- 実際の使用例
```

### 4. 保存と共有

```
.claude/learnings/
├── 2024-01-20-auth-pattern.md
├── 2024-01-21-api-error-handling.md
└── index.md  # 学習インデックス
```

## 学習ファイルフォーマット

```markdown
# [パターン名]

## 抽出日

2024-01-20

## コンテキスト

- プロジェクト: [プロジェクト名]
- 技術スタック: [使用技術]
- 課題: [解決した課題]

## パターン

### 概要

[パターンの概要説明]

### 実装

\`\`\`typescript
// コード例
\`\`\`

## 適用条件

- [条件 1]
- [条件 2]

## 関連パターン

- [関連パターン 1]
- [関連パターン 2]
```

## 自動学習トリガー

### Stop フックでの自動実行

セッション終了時に自動的にパターン抽出を実行：

1. 変更されたファイルの分析
2. 使用されたコマンドの分析
3. 解決された問題の特定
4. パターンの抽出と保存

## 学習の活用

### プロジェクト固有のスキル化

頻繁に使用されるパターンはプロジェクト固有のスキルとして登録：

```
.claude/skills/
└── project-patterns/
    └── SKILL.md
```

### チーム共有

学習したパターンをチームで共有：

- PR でのドキュメント追加
- Wiki への登録
- チーム MTG での共有
