# Coding Style Rules

## 【CRITICAL】ハードコーディング禁止

- **マジックナンバー**: 数値リテラルの直接記述は禁止
- **設定値**: API キー、URL、パスは環境変数または config から読み込む
- **UI 文字列**: エラーメッセージ、ラベルは定数化

## ファイルサイズ制限

- **推奨**: 200-400 行/ファイル
- **最大**: 500 行を超えたら分割を検討
- **関数**: 50 行以下を推奨

## イミュータビリティ

- `const` をデフォルトで使用
- オブジェクト/配列の変更は新しいインスタンスを作成
- `Object.freeze()` で重要なオブジェクトを保護

## 命名規約

- **変数/関数**: camelCase
- **クラス/型**: PascalCase
- **定数**: UPPER_SNAKE_CASE
- **ファイル**: kebab-case

## 構造化

- 早期リターン/ガード節を使用
- 深いネスト（3 レベル以上）を避ける
- 三項演算子は単純な場合のみ

## TypeScript

- `any` 使用禁止（`unknown` を使用）
- 明示的な戻り値の型を指定
- `strict: true` を有効化
